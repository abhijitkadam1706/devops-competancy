## What Is This?

This folder contains all the Terraform code that builds the AWS cloud infrastructure for the mern-auth application. It is the "blueprint" that creates your VPC, EKS cluster, DocumentDB, load balancer, IAM roles, and monitoring.

## Before You Start

You need these tools installed on your Windows machine:
- **AWS CLI** — `choco install awscli` (or download `.msi` from aws.amazon.com)
- **Terraform >= 1.7** — `choco install terraform`
- **kubectl** — `choco install kubernetes-cli`

Configure your AWS credentials:
```powershell
aws configure
# Enter: Access Key, Secret Key, Region (ap-southeast-1), output format (json)
```

## Step-by-Step Deployment

### Step 1: Create the State Bucket (Run Once Only)

```powershell
cd terraform\global
terraform init -backend=false
terraform apply -target=aws_s3_bucket.terraform_state -target=aws_dynamodb_table.terraform_locks
```

Note the output bucket name. Open `environments\production\main.tf`, `environments\staging\main.tf`, and `environments\dev\main.tf` and replace `203848753188` with your actual **12-digit AWS Account ID**.

### Step 2: Deploy Production Infrastructure

```powershell
cd ..\environments\production
terraform init
terraform apply
```

Type `yes` when prompted. Wait approximately 15-20 minutes.

### Step 3: Connect kubectl to Your New Cluster

```powershell
aws eks update-kubeconfig --region ap-southeast-1 --name mern-auth-prod
kubectl get nodes
```

## Folder Structure

```
terraform/
├── global/
│   ├── providers.tf      # AWS/Kubernetes provider version pinning
│   └── bootstrap.tf      # Creates S3 + DynamoDB for state storage
│
├── modules/              # Reusable building blocks — edit these for shared changes
│   ├── vpc/              # Network, subnets, NAT gateways, flow logs
│   ├── eks/              # Kubernetes cluster (private, encrypted)
│   ├── documentdb/       # MongoDB-compatible database (private, TLS)
│   ├── alb/              # Load balancer (public HTTPS entry point)
│   ├── iam/              # Security roles (least privilege, IRSA)
│   └── monitoring/       # CloudWatch alarms, SNS alerts, dashboard
│
└── environments/         # Per-environment settings
    ├── production/        # ON_DEMAND nodes, 3 DocDB instances, deletion protection
    ├── staging/           # SPOT nodes, 1 DocDB instance, cheaper
    └── dev/               # 1 SPOT node, no ALB, access via port-forward
```

## Variables You Need to Update

Open `environments/production/terraform.tfvars`:

| Variable | What to change |
|----------|----------------|
| `domain_name` | Your domain if you have one. Otherwise leave as `""` |
| `alert_email` | Your email for AWS alarm alerts. Leave as `""` to skip |

That is all. No secrets go in `.tfvars` — passwords are auto-generated by Terraform and stored in AWS Secrets Manager.
